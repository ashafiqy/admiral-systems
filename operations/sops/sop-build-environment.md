# Build Environment Policy

## Purpose
Protect Admiral Systems from losing access to unfinished client work by ensuring all builds happen in Admiral-controlled environments.

## When to Use
Every new client project that involves Webflow (or any build platform where workspace ownership matters). Apply from project kickoff.

## Steps
1. Create the project in Admiral's Webflow workspace — never in the client's workspace.
2. Complete all development work within Admiral's environment.
3. Share only the staging link with the client for review — do not grant workspace access.
4. Transfer the project to the client's workspace only after payment is confirmed and the project is complete.
5. If a project was started in a client's workspace, migrate it to Admiral's workspace before continuing work. If migration isn't possible, escalate to Davis via Shafiq.

## Notes
- This policy was established after a Wallex discussion on 6 Apr 2026 where the risk of losing access to in-progress work was identified.
- Applies to all engagement types: Beta, Epsilon, Retainer, and Pro Bono.
